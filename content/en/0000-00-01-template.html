Benefits of .Rmd file

  1. Run R code. The outputs can be automatically included and labeled.

    This feature is also supported in .Rmarkdown file.

  2. Math supported by Pandoc and MathJax.

    Inline math: \(e^{i\pi} + 1 = 0\) and display math:

    \[ \mathcal{L}_T(\vec{\lambda}) = \sum_{(x,s)\in T} \log P(s|X) - \sum_{i=1}^m \lambda_i^2 + \tfrac{a^2 + b^2}{c^3} + \nabla h - \ll s,t \gg \]

  3. Use \begin{equation} and reference.

    \[\begin{equation} e^{i\pi} + 1 = 0 \tag{1} \end{equation}\]

    You can reference Equation (1).

  4. Theorem, Definition, Lemma, and Proof environments

    Theorem 1 Here is a theorem.
    Proof. Trivia.

    We can reference Theorem 1. See bookdown for more details.

    Environment Label Prefix
    theorem thm
    lemma lem
    corollary cor
    proposition prp
    conjecture cnj
    definition def
    example exm
    exercise exr

  5. Syntax highlight works well with this blog template.

Cons of .Rmd file

No to-do list. And the file outputs are not that clean as .md file.
